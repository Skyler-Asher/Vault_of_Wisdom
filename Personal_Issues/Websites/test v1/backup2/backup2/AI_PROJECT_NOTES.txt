Project Overview
This folder contains a small local personal planner web app. It is built with plain HTML, CSS, JavaScript, and a tiny Python server. There is no framework, package manager, or build step.

Main files:
- index.html: the main day planner/checklist page.
- bank.html: the reusable task bank page.
- storage.js: shared browser-side storage helper.
- server.py: local Python server that serves the app and handles saved data.
- data.json: persistent app data.
- start.bat: Windows launcher that starts the server and opens the app.

How To Run
Use start.bat, or run server.py directly with Python from this folder.

The app is served at:
http://localhost:8000/index.html

Data Flow
The app saves state through storage.js.

storage.js exposes:
- store.get(key)
- store.set(key, value)
- store.remove(key)

These functions call the local server's /data endpoint. The server reads and writes data.json.

Important saved keys in data.json:
- monday_schedule_v2: checked/unchecked state for day tasks.
- day_order_v1: saved section order and task order.
- day_sections_v1: custom section labels.
- task_bank_v1: reusable tasks created in the task bank.
- day_custom_tasks_v1: tasks added from the task bank onto the day page.

Main App Behavior
index.html contains the day planner. It supports:
- checkable task cards
- progress bar
- date picker
- collapsible sections
- edit mode
- dragging/reordering sections and tasks with SortableJS
- deleting sections and tasks
- adding new sections
- reset modal
- importing tasks from the task bank

bank.html contains the task bank. It supports:
- creating reusable tasks
- assigning task name, tag/duration, note, icon, and color
- deleting saved bank tasks
- adding a bank task to the day page

Task Bank Import Behavior
When a task is added from bank.html to index.html, it is passed through the URL as a bank_task parameter.

The day page then:
1. reads the task from the URL
2. creates a day task card
3. saves it into day_custom_tasks_v1
4. removes the URL parameter so refresh does not duplicate the task

Important: day_custom_tasks_v1 is required so imported task-bank tasks persist after refresh or app restart.

Several save operations are asynchronous. If adding features that save data and immediately reload or navigate, wait for store.set/store.remove to finish where needed.
